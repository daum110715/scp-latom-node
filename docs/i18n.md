# Internationalization (i18n)

The app uses [vue-i18n](https://vue-i18n.intlify.dev/) for internationalization. Currently supported locales:

- **English** (`en`) ŌĆö default and fallback
- **Chinese** (`zh`)

Locale is persisted in `localStorage` under key `scp-locale`. On first visit, the browser language is detected automatically.

## Architecture

### Setup

| File | Purpose |
| ---- | ------- |
| `src/i18n.ts` | Creates the vue-i18n instance (Composition API mode, `legacy: false`) |
| `src/locales/en.ts` | English translation object |
| `src/locales/zh.ts` | Chinese translation object |
| `src/composables/useLocale.ts` | Composable for toggling/setting locale |

### Locale Detection Priority

1. `localStorage('scp-locale')` ŌĆö if previously set by the user
2. `navigator.language` ŌĆö if it starts with `'zh'`, use `zh`
3. Fallback: `'en'`

## Translation Structure

Translations are organized by feature domain:

| Namespace | Content |
| --------- | ------- |
| `site.*` | Site-wide strings (title, tagline) |
| `nav.*` | Navigation labels |
| `header.*` | Header UI strings |
| `sidebar.*` | Sidebar content |
| `footer.*` | Footer content |
| `hero.*` | Homepage hero section |
| `stats.*` | Statistics labels |
| `recent.*` | Recent entries section |
| `catalog.*` | Catalog page |
| `entry.*` | Entry detail page |
| `documents.*` | Documents page |
| `about.*` | About page |
| `notFound.*` | 404 page |
| `search.*` | Search modal |
| `classes.*` | Object class names |
| `classification.*` | Classification level names |
| `entries.*` | SCP entry content (name, summary, containment, description, addenda) |
| `docs.*` | Document content (title, summary, content) |
| `auth.*` | Authentication UI strings |
| `errors.*` | Error messages (mapped from `ErrorCode` enum) |

## Adding a New Language

1. Create `src/locales/<lang>.ts` following the structure of `en.ts`
2. Import it in `src/i18n.ts` and add to the `messages` object:

```ts
import zh from './locales/zh'
import fr from './locales/fr'  // new

const i18n = createI18n({
  legacy: false,
  locale: defaultLocale,
  fallbackLocale: 'en',
  messages: { en, zh, fr },
})
```

3. Add the locale to the detection logic in `src/i18n.ts` if needed
4. Update the language switcher in `AppHeader.vue` if adding a new option
5. Translate all keys ŌĆö missing keys automatically fall back to English

## Adding New Translation Keys

1. Add the key to `src/locales/en.ts` (English is the source of truth)
2. Add the corresponding key to `src/locales/zh.ts`
3. Use in components with the `$t()` function or `useI18n()` composable:

```vue
<template>
  <!-- Template usage -->
  <h1>{{ $t('catalog.title') }}</h1>
  <p>{{ $t('catalog.entriesFound', { count: 42 }) }}</p>
</template>

<script setup lang="ts">
import { useI18n } from 'vue-i18n'
const { t } = useI18n()

// Script usage
const title = t('catalog.title')
</script>
```

4. Use parameterized messages for dynamic content:

```ts
// en.ts
catalog: {
  entriesFound: '{count} entries found'
}

// zh.ts
catalog: {

}
```

## Error Message i18n

Error codes in `src/services/errors.ts` map to translation keys under the `errors.*` namespace. The `resolveErrorMessage()` function uses `useI18n()` to translate error codes at runtime.

| HTTP Status | Error Code | Translation Key |
| ----------- | ---------- | --------------- |
| 400 | `ERR-400-REQUEST` | `errors.ERR-400-REQUEST` |
| 401 | `ERR-401-CLEARANCE` | `errors.ERR-401-CLEARANCE` |
| 403 | `ERR-403-ACCESS` | `errors.ERR-403-ACCESS` |
| 404 | `ERR-404-RESOURCE` | `errors.ERR-404-RESOURCE` |
| 409 | `ERR-409-CONFLICT` | `errors.ERR-409-CONFLICT` |
| 429 | `ERR-429-THROTTLE` | `errors.ERR-429-THROTTLE` |
| 500 | `ERR-500-SYSTEM` | `errors.ERR-500-SYSTEM` |
| 502/503/504 | `ERR-503-MAINTENANCE` | `errors.ERR-503-MAINTENANCE` |

Fallback: if i18n context is unavailable, the raw error code string is returned.

## Best Practices

- **Always add keys to both locale files** in the same PR
- **Use parameterized messages** for dynamic content (e.g., `{count} entries found`)
- **Keep translation values as plain strings** ŌĆö avoid HTML in translations
- **Use consistent SCP terminology**: codename, clearance, containment, object class
- **Test both locales** when adding new UI text
